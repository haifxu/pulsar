RabbitMQ source connector

RabbitMQ 源连接器从 RabbitMQ 集群接收消息并向 Pulsar 主题写消息。

Configuration

RabbitMQ 源连接器的配置具有以下属性。

属性

配置项类型Required默认值说明
connectionNameStringtrue" "（空字符串）连接名称。
hostStringtrue" "（空字符串）RabbitMQ 主机。
portinttrue5672RabbitMQ 端口。
virtualHostStringtrue/连接到 RabbitMQ 的虚拟主机。
usernameStringfalseguestThe username used to authenticate to RabbitMQ.
passwordStringfalseguestThe password used to authenticate to RabbitMQ.
queueNameStringtrue" "（空字符串）消息应该读取或写到的 RabbitMQ 队列。
requestedChannelMaxintfalse0初始请求的最大信道数。

0 意味着无限制。
requestedFrameMaxintfalse0最初请求的最大通信帧大小，以字节为单位。

0 意味着无限制。
connectionTimeoutintfalse60000TCP 连接的超时时间（毫秒）。

0 意味着无限值。
handshakeTimeoutintfalse10000AMQP0-9-1 协议握手超时(毫秒)。
requestedHeartbeatintfalse60The requested heartbeat timeout in seconds.
prefetchCountintfalse0服务器发送的消息的最大数量。

0 意味着无限制。
prefetchGlobalbooleanfalsefalse设置是否应用于整个信道而不是每个消费者。
passivebooleanfalsefalseRabbitMQ 消费者是否创建自己的队列还是绑定到已有队列。

示例

在使用 RabbitMQ 源连接器之前，你需要通过以下方法之一创建一个配置文件。

  • JSON

    {
    "host": "localhost",
    "port": "5672",
    "virtualHost": "/",
    "username": "guest",
    "password": "guest",
    "queueName": "test-queue",
    "connectionName": "test-connection",
    "requestedChannelMax": "0",
    "requestedFrameMax": "0",
    "connectionTimeout": "60000",
    "handshakeTimeout": "10000",
    "requestedHeartbeat": "60",
    "prefetchCount": "0",
    "prefetchGlobal": "false",
    "passive": "false"
}

  • YAML

    configs:
    host: "localhost"
    port: 5672
    virtualHost: "/"
    username: "guest"
    password: "guest"
    queueName: "test-queue"
    connectionName: "test-connection"
    requestedChannelMax: 0
    requestedFrameMax: 0
    connectionTimeout: 60000
    handshakeTimeout: 10000
    requestedHeartbeat: 60
    prefetchCount: 0
    prefetchGlobal: "false"
    passive: "false"
