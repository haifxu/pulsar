Debezium source connector

Debezium 源连接器将消息从 MySQL 或 PostgreSQL 提取并持久化到 Pulsar 主题。

Configuration

Debezium 源连接器的配置具有以下属性。

配置项Required默认值说明
task.classtruenullA source task class that implemented in Debezium.
database.hostnametruenull数据库服务器地址。
database.porttruenull数据库服务器的端口号。
database.usertruenull具有所需权限的数据库用户名称。
database.passwordtruenull具有所需权限的数据库用户的密码。
database.server.idtruenull数据库集群中的连接器标识符必须唯一，与数据库的服务器 Id（server-id）配置属性类似。
database.server.nametruenull数据库服务器/集群的逻辑名称，它构成命名空间并用于连接器写入的 Kafka 主题的所有名称，Kafka Connect schema 名称和使用 Avro 连接器时相应的 Avro schema 的命名空间。
database.whitelistfalsenull由此服务器托管的所有数据库的列表，并处于连接器监测中。

这是可选的，还有其他属性用于列出数据库和表格以包含或排除在监视之外。
key.convertertruenullThe converter provided by Kafka Connect to convert record key.
value.convertertruenullThe converter provided by Kafka Connect to convert record value.
database.historytruenull数据库历史类的名称。
database.history.pulsar.topictruenull数据库历史主题的名称，连接器在这里写入和恢复 DDL 语句。

注意：本主题仅供内部使用，外部消费者不应使用。
database.history.pulsar.service.urltruenull历史主题的 Pulsar 集群服务 URL
pulsar.service.urltruenullPulsar 群集服务 URL。
offset.storage.topictruenull记录连接器成功完成的最后一次提交 offset。
json-with-envelopefalsefalse展示的消息只包含 payload。

转换器的选项

  1. org.apache.kafka.connect.json.JsonConverter

这个配置 json-with-envelope 只对 JsonConverter 有效。 该配置默认值为 false，消费者使用 Schema.KeyValue(Schema.AUTO_CONSUME(), Schema.AUTO_CONSUME(), KeyValueEncodingType.SEPARATED) schema，消息中只包含 payload。

如果配置 json-with-envelope 值为 true，消费者将使用 Schema.KeyValue(Schema.BYTES, Schema.BYTES schema， 消息由 schema 和 payload 组成。

  1. org.apache.pulsar.kafka.shade.io.confluent.connect.avro.AvroConverter

如果用户选择 AvroConverter，那么 pulsar 消费者应该使用 Schema.KeyValue(Schema.AUTO_CONSUME(),Schema.AUTO_CONSUME(), KeyValueEncodingType.SEPARATED) schema，消息由 payload 组成。

MongoDB Configuration

配置项Required默认值说明
mongodb.hoststruenull复制集中的 MongoDB 服务器主机名和端口对（形式为 'host' 或 'host:port'）列表，用逗号分隔。 列表包含单个主机名和端口对。 如果 mongodb.members.auto.discover 设置为 false，主机和端口对将以复制集合名称为前缀（例如，rs0/localhost:27017)。
mongodb.nametruenull标识连接器和/或 MongoDB 复制器集合，或该连接器监控的共享集群的唯一名称。 每台服务器应该最多受到一个 Debezium 连接器的监控，因为这个服务器名称是所有来自 MongoDB 副本集或集群的持久 Kafka 主题的前缀。
mongodb.usertruenull连接到 MongoDB 时要使用的数据库用户名称。 只有当 MongoDB 被配置为使用身份验证时才需要这么做。
mongodb.passwordtruenull连接到 MongoDB 时使用的密码。 只有当 MongoDB 被配置为使用身份验证时才需要这么做。
mongodb.task.idtruenullMongoDB 连接器的 taskId，为每个复制集使用一个单独的任务。

Example of MySQL

你需要在使用 Pulsar Debezium 连接器之前创建一个配置文件。

Configuration

你可以使用以下方法之一来创建配置文件。

  • JSON

    {
    "database.hostname": "localhost",
    "database.port": "3306",
    "database.user": "debezium",
    "database.password": "dbz",
    "database.server.id": "184054",
    "database.server.name": "dbserver1",
    "database.whitelist": "inventory",
    "database.history": "org.apache.pulsar.io.debezium.PulsarDatabaseHistory",
    "database.history.pulsar.topic": "history-topic",
    "database.history.pulsar.service.url": "pulsar://127.0.0.1:6650",
    "key.converter": "org.apache.kafka.connect.json.JsonConverter",
    "value.converter": "org.apache.kafka.connect.json.JsonConverter",
    "pulsar.service.url": "pulsar://127.0.0.1:6650",
    "offset.storage.topic": "offset-topic"
}

  • YAML

    你可以创建一个 debezium-mysql-source-config.yaml 文件，并将下面的 内容 复制到 debezium-mysql-source-config.yaml 文件。

    tenant: "public"
namespace: "default"
name: "debezium-mysql-source"
topicName: "debezium-mysql-topic"
archive: "connectors/pulsar-io-debezium-mysql-2.9.1.nar"
parallelism: 1

configs:

    ## config for mysql, docker image: debezium/example-mysql:0.8
    database.hostname: "localhost"
    database.port: "3306"
    database.user: "debezium"
    database.password: "dbz"
    database.server.id: "184054"
    database.server.name: "dbserver1"
    database.whitelist: "inventory"
    database.history: "org.apache.pulsar.io.debezium.PulsarDatabaseHistory"
    database.history.pulsar.topic: "history-topic"
    database.history.pulsar.service.url: "pulsar://127.0.0.1:6650"

    ## KEY_CONVERTER_CLASS_CONFIG, VALUE_CONVERTER_CLASS_CONFIG
    key.converter: "org.apache.kafka.connect.json.JsonConverter"
    value.converter: "org.apache.kafka.connect.json.JsonConverter"

    ## PULSAR_SERVICE_URL_CONFIG
    pulsar.service.url: "pulsar://127.0.0.1:6650"

    ## OFFSET_STORAGE_TOPIC_CONFIG
    offset.storage.topic: "offset-topic"

用法

此示例显示如何使用 Pulsar Debezium 连接器更改 MySQL 的数据。

  1. 启动一个 MySQL 服务器，Debezium 可以从该数据库捕获更改。

    $ docker run -it --rm \
--name mysql \
-p 3306:3306 \
-e MYSQL_ROOT_PASSWORD=debezium \
-e MYSQL_USER=mysqluser \
-e MYSQL_PASSWORD=mysqlpw debezium/example-mysql:0.8

  2. 以独立模式启动本地的 Pulsar 服务。

    $ bin/pulsar standalone

  3. 在本地运行模式中使用以下的一种方法启动 Pulsar Debezium 连接器。

    • 使用前面显示的 JSON 配置文件。

      请确保 nar 文件可在 connector/pulsar-io-debezium-mysql-2.9.1.nar 上使用。

      $ bin/pulsar-admin source localrun \
--archive connectors/pulsar-io-debezium-mysql-2.9.1.nar \
--name debezium-mysql-source --destination-topic-name debezium-mysql-topic \
--tenant public \
--namespace default \
--source-config '{"database.hostname": "localhost","database.port": "3306","database.user": "debezium","database.password": "dbz","database.server.id": "184054","database.server.name": "dbserver1","database.whitelist": "inventory","database.history": "org.apache.pulsar.io.debezium.PulsarDatabaseHistory","database.history.pulsar.topic": "history-topic","database.history.pulsar.service.url": "pulsar://127.0.0.1:6650","key.converter": "org.apache.kafka.connect.json.JsonConverter","value.converter": "org.apache.kafka.connect.json.JsonConverter","pulsar.service.url": "pulsar://127.0.0.1:6650","offset.storage.topic": "offset-topic"}'

    • 使用前面显示的 YAML 配置文件。

      $ bin/pulsar-admin source localrun \
--source-config-file debezium-mysql-source-config.yaml

  4. 订阅 _ sub 主题 表中的子产品_

    $ bin/pulsar-client consume -s "sub-products" public/default/dbserver1.inventory.products -n 0

  5. 在 docker 中启动 MySQL 客户端。

    $ docker run -it --rm \
--name mysqlterm \
--link mysql \
--rm mysql:5.7 sh \
-c 'exec mysql -h"$MYSQL_PORT_3306_TCP_ADDR" -P"$MYSQL_PORT_3306_TCP_PORT" -uroot -p"$MYSQL_ENV_MYSQL_ROOT_PASSWORD"'

  6. 弹出 MySQL 客户端。

    使用以下命令更改表 产品 的数据。

    mysql> use inventory;
mysql> show tables;
mysql> SELECT * FROM  products;
mysql> UPDATE products SET name='1111111111' WHERE id=101;
mysql> UPDATE products SET name='1111111111' WHERE id=107;

    在订阅主题的终端窗口中，你可以在 子产品 主题中找到数据变更。

PostgreSQL 示例

你需要在使用 Pulsar Debezium 连接器之前创建一个配置文件。

Configuration

你可以使用以下方法之一来创建配置文件。

  • JSON

    {
    "database.hostname": "localhost",
    "database.port": "5432",
    "database.user": "postgres",
    "database.password": "changeme",
    "database.dbname": "postgres",
    "database.server.name": "dbserver1",
    "plugin.name": "pgoutput",
    "schema.whitelist": "public",
    "table.whitelist": "public.users",
    "pulsar.service.url": "pulsar://127.0.0.1:6650"
}

  • YAML

    你可以创建一个 debezium-postgres-config.yaml 文件，并将下面的 内容 复制到 debezium-postgres-source-config.yaml 文件。

    tenant: "public"
namespace: "default"
name: "debezium-postgres-source"
topicName: "debezium-postgres-topic"
archive: "connectors/pulsar-io-debezium-postgres-2.9.1.nar"
parallelism: 1

configs:

    ## config for postgres version 10+, official docker image: postgres:<10+>
    database.hostname: "localhost"
    database.port: "5432"
    database.user: "postgres"
    database.password: "changeme"
    database.dbname: "postgres"
    database.server.name: "dbserver1"
    plugin.name: "pgoutput"
    schema.whitelist: "public"
    table.whitelist: "public.users"

    ## PULSAR_SERVICE_URL_CONFIG
    pulsar.service.url: "pulsar://127.0.0.1:6650"

请注意，pgoutput 是 Postgres 版本 10 中引入的标准插件 - 请参阅 Postgres 架构文档。 你不需要安装任何东西，只需确保 WAL 级别设置为 logical(请参阅下面的 docker 命令和 Postgres 文档)。

用法

此示例显示如何使用 Pulsar Debezium 连接器更改 PostgreSQL 表的数据。

  1. 启动一个 PostgreSQL 服务器，Debezium 可以从该数据库捕获更改。

    $ docker run -d -it --rm \
--name pulsar-postgres \
-p 5432:5432 \
-e POSTGRES_PASSWORD=changeme \
postgres:13.3 -c wal_level=logical

  2. 以独立模式启动本地的 Pulsar 服务。

    $ bin/pulsar standalone

  3. 在本地运行模式中使用以下的一种方法启动 Pulsar Debezium 连接器。

    • 使用前面显示的 JSON 配置文件。

      Make sure the nar file is available at connectors/pulsar-io-debezium-postgres-2.9.1.nar.

      $ bin/pulsar-admin source localrun \
--archive connectors/pulsar-io-debezium-postgres-2.9.1.nar \
--name debezium-postgres-source \
--destination-topic-name debezium-postgres-topic \
--tenant public \
--namespace default \
--source-config '{"database.hostname": "localhost","database.port": "5432","database.user": "postgres","database.password": "changeme","database.dbname": "postgres","database.server.name": "dbserver1","schema.whitelist": "public","table.whitelist": "public.users","pulsar.service.url": "pulsar://127.0.0.1:6650"}'

    • 使用前面显示的 YAML 配置文件。

      $ bin/pulsar-admin source localrun  \
--source-config-file debezium-postgres-source-config.yaml

  4. 订阅 sub-users 主题表中的 public.users。

    $ bin/pulsar-client consume -s "sub-users" public/default/dbserver1.public.users -n 0

  5. 在 Docker 中启动一个 PostgreSQL 客户端。

    $ docker exec -it pulsar-postgresql /bin/bash

  6. 弹出 PostgreSQL 客户端。

    使用以下命令在 users 表中创建样本数据。

    psql -U postgres -h localhost -p 5432

Example of MongoDB

你需要在使用 Pulsar Debezium 连接器之前创建一个配置文件。

  • JSON

    {
    "mongodb.hosts": "rs0/mongodb:27017",
    "mongodb.name": "dbserver1",
    "mongodb.user": "debezium",
    "mongodb.password": "dbz",
    "mongodb.task.id": "1",
    "database.whitelist": "inventory",
    "pulsar.service.url": "pulsar://127.0.0.1:6650"

    You can create a debezium-mongodb-source-config.yaml file and copy the contents below to the debezium-mongodb-source-config.yaml file.

    tenant: "public"

    ## config for pg, docker image: debezium/example-mongodb:0.10

    database.whitelist: "inventory",

    ## PULSAR_SERVICE_URL_CONFIG
    pulsar.service.url: "pulsar://127.0.0.1:6650"

用法

This example shows how to change the data of a MongoDB table using the Pulsar Debezium connector.

  1. 启动一个 MongoDB 服务器，Debezium 可以从该数据库捕获更改。

    $ docker pull debezium/example-mongodb:0.10
$ docker run -d -it --rm --name pulsar-mongodb -e MONGODB_USER=mongodb -e MONGODB_PASSWORD=mongodb -p 27017:27017  debezium/example-mongodb:0.10

    Use the following commands to initialize the data.

    ./usr/local/bin/init-inventory.sh

    If the local host cannot access the container network, you can update the file /etc/hosts and add a rule 127.0.0.1 6 f114527a95f. f114527a95f is container id, you can try to get by docker ps -a

  1. 以独立模式启动本地的 Pulsar 服务。

    $ bin/pulsar standalone

  2. 在本地运行模式中使用以下的一种方法启动 Pulsar Debezium 连接器。

    • 使用前面显示的 JSON 配置文件。

      Make sure the nar file is available at connectors/pulsar-io-mongodb-2.9.1.nar.

      $ bin/pulsar-admin source localrun \
--archive connectors/pulsar-io-debezium-mongodb-2.9.1.nar \
--name debezium-mongodb-source \
--destination-topic-name debezium-mongodb-topic \
--tenant public \
--namespace default \
--source-config '{"mongodb.hosts": "rs0/mongodb:27017","mongodb.name": "dbserver1","mongodb.user": "debezium","mongodb.password": "dbz","mongodb.task.id": "1","database.whitelist": "inventory","pulsar.service.url": "pulsar://127.0.0.1:6650"}'

    • 使用前面显示的 YAML 配置文件。

      $ bin/pulsar-admin source localrun  \
--source-config-file debezium-mongodb-source-config.yaml

  3. 为 sub 产品 订阅表中的主题 子产品。

    $ bin/pulsar-client consume -s "sub-products" public/default/dbserver1.inventory.products -n 0

  4. 在 docker 中启动 MongoDB 客户端。

    $ docker exec -it pulsar-mongodb /bin/bash

  5. 弹出 MongoDB 客户端。

    mongo -u debezium -p dbz --authenticationDatabase admin localhost:27017/inventory
db.products.update({"_id":NumberLong(104)},{$set:{weight:1.25}})
