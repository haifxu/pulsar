JDBC sink connector

JDBC sink 连接器允许从 Pulsar 主题拉取消息，并将消息持久化到 ClickHouse、MariaDB、PostgreSQL 或 SQlite。

目前，支持 INSERT、DELETE 和 UPDATE 操作。

Configuration

JDBC sink 连接器的配置具有以下属性。

属性

配置项类型Required默认值说明
userNameStringfalse" "（空字符串）The username used to connect to the database specified by jdbcUrl.

Note: userName is case-sensitive.
passwordStringfalse" "（空字符串）用于连接到 jdbcUrl 指定的数据库的密码。

Note: password is case-sensitive.
jdbcUrlStringtrue" "（空字符串）连接器连接的数据库的 JDBC URL。
tableNameStringtrue" "（空字符串）连接器写入的表名。
nonKeyStringfalse" "（空字符串）逗号分隔的列表，包含用于更新事件的字段。
keyStringfalse" "（空字符串）逗号分隔的列表，包含在 where 更新和删除事件的条件的字段。
timeoutMsintfalse500JDBC 操作超时，单位为毫秒。
batchSizeintfalse200The batch size of updates made to the database.

ClickHouse 示例

  • JSON

    {
    "userName": "clickhouse",
    "password": "password",
    "jdbcUrl": "jdbc:clickhouse://localhost:8123/pulsar_clickhouse_jdbc_sink",
    "tableName": "pulsar_clickhouse_jdbc_sink"
}

  • YAML

    tenant: "public"
namespace: "default"
name: "jdbc-clickhouse-sink"
topicName: "persistent://public/default/jdbc-clickhouse-topic"
sinkType: "jdbc-clickhouse"    
configs:
    userName: "clickhouse"
    password: "password"
    jdbcUrl: "jdbc:clickhouse://localhost:8123/pulsar_clickhouse_jdbc_sink"
    tableName: "pulsar_clickhouse_jdbc_sink"

PostgreSQL 示例

在使用 JDBC PostgreSQL sink 连接器之前，你需要通过以下方法之一创建一个配置文件。

  • JSON

    {
    "userName": "postgres",
    "password": "password",
    "jdbcUrl": "jdbc:postgresql://localhost:5432/pulsar_postgres_jdbc_sink",
    "tableName": "pulsar_postgres_jdbc_sink"
}

  • YAML

    tenant: "public"
namespace: "default"
name: "jdbc-postgres-sink"
topicName: "persistent://public/default/jdbc-postgres-topic"
sinkType: "jdbc-postgres"    
configs:
    userName: "postgres"
    password: "password"
    jdbcUrl: "jdbc:postgresql://localhost:5432/pulsar_postgres_jdbc_sink"
    tableName: "pulsar_postgres_jdbc_sink"

关于 如何使用 JDBC sink 连接器 的更多信息，请参阅 将 Pulsar 连接到 PostgreSQL 。
