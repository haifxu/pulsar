HDFS2 sink connector

HDFS2 sink 连接器将消息从 Pulsar 主题 取出，并将消息持久化到 HDFS 文件中。

Configuration

HDFS2 sink 连接器的配置具有以下属性。

属性

配置项类型Required默认值说明
hdfsConfigResourcesStringtrue无包含 Hadoop 文件系统配置的文件，或以逗号分隔的列表。

示例
'core-site.xml'
'hdfs-site.xml'
directoryStringtrue无文件读取或写入的 HDFS 目录。
encodingStringfalse无文件的字符编码。

示例
UTF-8
ASCII
compressionCompressionfalse无用于压缩或解压缩 HDFS 上文件的压缩代码。

下面是可用的选项：
  • BZIP2
  • DEFLATE
  • GZIP
  • LZ4
  • SNAPPY
    • kerberosUserPrincipalStringfalse无用于身份验证的 Kerberos 用户主账户。
    keytabStringfalse无用于身份验证的 Kerberos keytab 文件的完整路径名。
    filenamePrefixStringtrue，如果 compression 设置为 None。无在 HDFS 目录中创建的文件的前缀。

    示例
    在名称为 topicA- 的文件里的 topicA 的值。
    fileExtensionStringtrue无扩展已添加到写入 HDFS 的文件。

    示例
    '.txt'
    '.seq'
    separatorcharfalse无用于在文本文件中分离记录的字符。

    如果没有提供任何值，所有记录中的内容都会在一个连续的字节数组中集中在一起。
    syncIntervallongfalse0数据刷新到 HDFS 磁盘的调用间隔（毫秒）
    maxPendingRecordsintfalseInteger.MAX_VALUEAck（确认）前在内存中保留的最大记录数。

    将此属性设置为 1，在记录确认前将每条记录发送到磁盘。

    将此属性设置为更高的值，则允许在刷新到磁盘之前缓存记录。
    subdirectoryPatternStringfalse无与 Sink 创建时间相关的子目录。
    此处的模式（Pattern）是指目录下子目录的格式化模式。

    模式语法详情，可参考DateTimeFormatter。

    示例

    在使用 HDFS2 sink 连接器之前，你需要通过以下方法之一创建一个配置文件。

    • JSON

      {
    "hdfsConfigResources": "core-site.xml",
    "directory": "/foo/bar",
    "filenamePrefix": "prefix",
    "fileExtension": ".log",
    "compression": "SNAPPY",
    "subdirectoryPattern": "yyyy-MM-dd"
}

    • YAML

      configs:
    hdfsConfigResources: "core-site.xml"
    directory: "/foo/bar"
    filenamePrefix: "prefix"
    fileExtension: ".log"
    compression: "SNAPPY"
    subdirectoryPattern: "yyyy-MM-dd"
