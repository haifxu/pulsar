Netty source connector

Netty 源连接器打开一个端口，用于接收借助配置的网络协议传入的数据， 并将其发布到用户定义的 Pulsar 主题上。

此连接器可以用在容器部署中（比如 k8s）。 否则，如果连接器以进程或线程模式运行，实例可能在监听端口时发生冲突。

Configuration

Netty 源连接器的配置具有以下属性。

属性

配置项类型Required默认值说明
typeStringtruetcp数据传输到 Netty 的网络协议。

以下是可用的选项：
  • tcp
  • http
  • udp
    • hostStringtrue127.0.0.1源实例监听的主机名或地址。
    portinttrue10999源实例监听的端口。
    numberOfThreadsinttrue1Netty TCP 服务器接受传入连接和处理已接受连接流量的线程数。

    示例

    在使用 Netty 源连接器之前，需要通过以下方法之一创建一个配置文件。

    • JSON

      {
    "type": "tcp",
    "host": "127.0.0.1",
    "port": "10911",
    "numberOfThreads": "1"
}

    • YAML

      configs:
    type: "tcp"
    host: "127.0.0.1"
    port: 10999
    numberOfThreads: 1

    用法

    下面的示例显示如何使用 TCP 和 HTTP 的 Netty 源连接器。

    TCP

    1. 启动 Pulsar 单机模式。

      $ docker pull apachepulsar/pulsar:{version}

$ docker run -d -it -p 6650:6650 -p 8080:8080 -v $PWD/data:/pulsar/data --name pulsar-netty-standalone apachepulsar/pulsar:{version} bin/pulsar standalone

    2. 创建配置文件 nety-source-config.yaml。

      configs:
    type: "tcp"
    host: "127.0.0.1"
    port: 10999
    numberOfThreads: 1

    3. 复制配置文件 nety-source-config.yaml 到 Pulsar 服务器。

      $ docker cp netty-source-config.yaml pulsar-netty-standalone:/pulsar/conf/

    4. 下载 Netty 源连接器。

      $ docker exec -it pulsar-netty-standalone /bin/bash
curl -O http://mirror-hk.koddos.net/apache/pulsar/pulsar-{version}/connectors/pulsar-io-netty-{version}.nar

    5. 启动 Netty 源连接器。

      $ ./bin/pulsar-admin sources localrun \
--archive pulsar-io-2.9.1.nar \
--tenant public \
--namespace default \
--name netty \
--destination-topic-name netty-topic \
--source-config-file netty-source-config.yaml \
--parallelism 1

    6. 消费数据。

      $ docker exec -it pulsar-netty-standalone /bin/bash

$ ./bin/pulsar-client consume -t Exclusive -s netty-sub netty-topic -n 0

    7. 打开另一个终端窗口将数据发送到 Netty 源。

      $ docker exec -it pulsar-netty-standalone /bin/bash

$ apt-get update

$ apt-get -y install telnet

$ root@1d19327b2c67:/pulsar# telnet 127.0.0.1 10999
Trying 127.0.0.1...
Connected to 127.0.0.1.
Escape character is '^]'.
hello
world

    8. 以下信息出现在消费者终端窗口。

      ----- got message -----
hello

----- got message -----
world

curl -O http://mirror-hk.koddos.net/apache/pulsar/pulsar-{version}/connectors/pulsar-io-netty-{version}.nar

$ curl -X POST --data 'hello, world!' http://127.0.0.1:10999/

    8. 以下信息出现在消费者终端窗口。

hello, world!
