Kafka sink connector

Kafka sink connector 从 Pulsar topic 中拉取消息，并将消息保存在 Kafka 中。

本文将会介绍如何配置和使用 Kafka sink connector。

Configuration

可以为 Kafka sink connector 配置以下参数。

属性

配置项类型Required默认值说明
bootstrapServersStringtrue" "（空字符串）用于创建与 Kafka 集群进行初始连接的主机和接口对列表（以逗号分隔）。
acksStringtrue" "（空字符串）Producer 要求 leader 在请求完成前收到的确认数量。
This controls the durability of the sent records.
batchsizelongfalse16384LKafka producer 将记录发送到 broker 前，整合批记录的批量大小。
maxRequestSizelongfalse1048576LKafka 请求的最大字节大小。
topicStringtrue" "（空字符串）从 Pulsar 接收消息的 Kafka topic。
keyDeserializationClassStringfalseorg.apache.kafka.common.serialization.StringSerializerKafka producer 用于序列化密钥的序列化器类。
valueDeserializationClassStringfalseorg.apache.kafka.common.serialization.ByteArraySerializerThe serializer class for Kafka producers to serialize values.

The serializer is set by a specific implementation of KafkaAbstractSink.
producerConfigPropertiesMapfalse" "（空字符串）要传递给 producer 的 producer 配置属性。

Note: other properties specified in the connector configuration file take precedence over this configuration.

示例

使用 Kafka sink connector 前，需要先创建一个配置文件，创建方法有以下两种。

  • JSON

    {
    "bootstrapServers": "localhost:6667",
    "topic": "test",
    "acks": "1",
    "batchSize": "16384",
    "maxRequestSize": "1048576",
    "producerConfigProperties":
     {
        "client.id": "test-pulsar-producer",
        "security.protocol": "SASL_PLAINTEXT",
        "sasl.mechanism": "GSSAPI",
        "sasl.kerberos.service.name": "kafka",
        "acks": "all" 
     }
}

  • YAML

    configs:
    bootstrapServers: "localhost:6667"
    topic: "test"
    acks: "1"
    batchSize: "16384"
    maxRequestSize: "1048576"
    producerConfigProperties:
        client.id: "test-pulsar-producer"
        security.protocol: "SASL_PLAINTEXT"
        sasl.mechanism: "GSSAPI"
        sasl.kerberos.service.name: "kafka"
        acks: "all"
