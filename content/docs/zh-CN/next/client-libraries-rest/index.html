Pulsar REST

Pulsar 不仅提供 REST 端点来管理 Pulsar 集群中的资源，而且还提供了查询这些资源状态的方法。 此外，Pulsar REST 提供了一种简单的方式来与 Pulsar 交互而不使用客户端库， 这让应用程序可以方便地使用 HTTP 与 Pulsar 交互。

连接

为了连接到 Pulsar，您需要指定一个 URL。

  • 向非分区主题或分区主题生产消息

    brokerUrl:{8080/8081}/topics/{persistent/non-persistent}/{my-tenant}/{my-namespace}/{my-topic}

  • 向分区主题的特定分区生产消息

    brokerUrl:{8080/8081}/topics/{persistent/non-persistent}/{my-tenant}/{my-namespace}/{my-topic}/partitions/{partition-number}

生产者（Producer）

目前，您可以使用 cURL 或 Postman 等工具通过 REST 方式向以下主题生产消息。

  • 未分区主题或分区主题

  • 分区主题的特定分区

Note

您只能通过 REST 向 Pulsar 上 已经存在的主题生产消息。

未来将会支持通过 REST 消费和读取消息。

消息（Message）

  • 下面是请求的结构。

    参数|是否必需?|描述 |---|---|--- schemaVersion|否| 该消息使用的 schema 的版本

    您需要提供以下内容之一：

    - schemaVersion
    - keySchema/valueSchema

    如果二者都提供了，则使用 schemaVersion 。 keySchema/valueSchema|否|该消息所使用的键 schema 或者值 schema producerName|否|生产者名称 Messages[] SingleMessage|是|所需发送的消息

  • 下面是消息的结构。

    参数|是否必需?|类型|描述 |---|---|---|--- payload|是|String|实际的消息内容

    消息使用字符串发送，并在服务端用给定的 schema 编码 properties|否|Map<String, String>|自定义属性 key|否|String|分区键 replicationClusters|否|List<String>| 消息要复制到的的集群 eventTime|否|String|消息事件时间 sequenceId|否|long|消息序列ID disableReplication|否|boolean|是否禁用消息复制 deliverAt|否|long|仅在指定的绝对时间戳时或之后分发消息 deliverAfterMs|否|long|仅在指定的相对延迟(以毫秒为单位)之后分发消息

Schema

  • 目前支持 Primitive、Avro、JSON 和 KeyValue 等 schema。

  • 对于 Primitive、Avro 和 JSON schema，必须提供 schema，并以字符串编码。

  • 如果未设置 schema，则消息将以字符串模式编码。
