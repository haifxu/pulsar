Pulsar Python 客户端

Pulsar 的 Python 客户端库是对现有 C++ 客户端库 的封装， 它所包含的功能与 C++ 客户端 相同。 你可以在 C++ 客户端源码的 Python 子目录 中找到 Pulsar Python 客户端的相关源码 。

Python 客户端中生产者、消费者和 Reader 的所有方法都是线程安全的。

关于 pdoc 生成的适用于 Python 客户端的 API 文档，可以参阅 这里。

安装

你可以通过两种方式安装 pulsar-client 库，一是通过 PyPi、使用 pip，或者从源码构建。

使用 pip 安装

使用包管理工具 pip 安装已经打包好的 pulsar-client：

$ pip install pulsar-client==2.9.1

可选依赖

如果你想在 Linux 上安装客户端来支持 Pulsar functions 或 Avro 序列化等服务， 可以在安装 pulsar-client 时安装相关可选组件。

# avro serialization
$ pip install pulsar-client[avro]=='2.9.1'

# functions runtime
$ pip install pulsar-client[functions]=='2.9.1'

# all optional components
$ pip install pulsar-client[all]=='2.9.1'

目前 PyPi 安装方式支持以下 Python 版本：

平台支持的 Python 版本
MacOS
10.13 (High Sierra), 10.14 (Mojave)2.7, 3.7, 3.8, 3.9
Linux2.7, 3.4, 3.5, 3.6, 3.7, 3.8, 3.9

从源代码安装

通过源码安装 pulsar-client，请参照安装步骤中的示例进行操作并编译 Pulsar C++ 客户端库。 这会构建 C++ 的 Python 封装包。

安装已经构建好的 Python 包：

$ git clone https://github.com/apache/pulsar
$ cd pulsar/pulsar-client-cpp/python
$ sudo python setup.py install

API 参考

完整的 Python API 参考，可参阅api/python。

示例

你可以找到多种 pulsar-client 库的 Python 代码示例。

生产者示例

下面示例展示为 my-topic 主题创建一个 Python 生产者，并向这个主题发送 10 条消息。

import pulsar

client = pulsar.Client('pulsar://localhost:6650')

producer = client.create_producer('my-topic')

for i in range(10):
    producer.send(('Hello-%d' % i).encode('utf-8'))

client.close()

消费者示例

下面示例展示在 my-topic 主题上创建带有 my-subscription 订阅名称的一个消费者，可用于接收传入消息、打印内容以及到达消息的 ID，并向 Pulsar broker 确认每条消息。

import pulsar

client = pulsar.Client('pulsar://localhost:6650')

consumer = client.subscribe('my-topic', 'my-subscription')

while True:
    msg = consumer.receive()
    try:
        print("Received message '{}' id='{}'".format(msg.data(), msg.message_id()))
        # Acknowledge successful processing of the message
        consumer.acknowledge(msg)
    except Exception:
        # Message failed to be processed
        consumer.negative_acknowledge(msg)

client.close()

该示例展示如何配置否定确认（negative acknowledgement）。

from pulsar import Client, schema
client = Client('pulsar://localhost:6650')
consumer = client.subscribe('negative_acks','test',schema=schema.StringSchema())
producer = client.create_producer('negative_acks',schema=schema.StringSchema())
for i in range(10):
    print('send msg "hello-%d"' % i)
    producer.send_async('hello-%d' % i, callback=None)
producer.flush()
for i in range(10):
    msg = consumer.receive()
    consumer.negative_acknowledge(msg)
    print('receive and nack msg "%s"' % msg.data())
for i in range(10):
    msg = consumer.receive()
    consumer.acknowledge(msg)
    print('receive and ack msg "%s"' % msg.data())
try:
    # No more messages expected
    msg = consumer.receive(100)
except:
    print("no more msg")
    pass

Reader 接口示例

你可以使用 Pulsar Python API 调用 Pulsar reader 接口。示例如下：

# MessageId 取自先前获取的消息
msg_id = msg.message_id()

reader = client.create_reader('my-topic', msg_id)

while True:
    msg = reader.read_next()
    print("Received message '{}' id='{}'".format(msg.data(), msg.message_id()))
    # 无确认操作

多主题订阅

除了向单一 Pulsar 主题订阅一个消费者外，你也可以同时订阅多个主题。 若要使用多主题订阅，可以提供一个正则表达式 (regex) 或主题List 。 如果通过 regex 选择主题, 则所有主题都必须位于同一Pulsar命名空间中。

如下所示：

import re
consumer = client.subscribe(re.compile('persistent://public/default/topic-*'), 'my-subscription')
while True:
    msg = consumer.receive()
    try:
        print("Received message '{}' id='{}'".format(msg.data(), msg.message_id()))
        # Acknowledge successful processing of the message
        consumer.acknowledge(msg)
    except Exception:
        # Message failed to be processed
        consumer.negative_acknowledge(msg)
client.close()

Schema

Supported schema types

Pulsar提供多种的序列化类型可供选择。所有序列化的类型都定义在 pulsar.schema 包中

Schema备注
BytesSchema默认 schema 模式，获取原始 payload 为 bytes 对象，不进行序列化或者反序列化。
StringSchema编码和解码的 payload 为 UTF-8 字符集。使用 str 对象
JsonSchema需要记录定义。将记录序列化为标准的 JSON payload
AvroSchema需要记录定义。序列化为 AVRO 格式。

Schema definition reference

The schema definition is done through a class that inherits from pulsar.schema.Record.

This class has a number of fields which can be of either pulsar.schema.Field type or another nested Record. All the fields are specified in the pulsar.schema package. The fields are matching the AVRO fields types.

字段类型Python 类型备注
Booleanbool
Integerint
Longint
Floatfloat
Doublefloat
Bytesbytes
Stringstr
Arraylist需要指定元素的记录类型。
MapdictKey 总是 String 类型，需要指定值类型。

Additionally, any Python Enum type can be used as a valid field type.

字段参数

When adding a field, you can use these parameters in the constructor.

参数默认值备注
default无设置字段默认值。例如：a = Integer(default=5)
requiredFalse标记该字段为“必填”，并设置相应的 schema。

Schema 定义示例

简单定义
class Example(Record):
    a = String()
    b = Integer()
    c = Array(String())
    i = Map(String())
使用枚举
from enum import Enum

class Color(Enum):
    red = 1
    green = 2
    blue = 3

class Example(Record):
    name = String()
    color = Color
复杂类型
class MySubRecord(Record):
    x = Integer()
    y = Long()
    z = String()

class Example(Record):
    a = String()
    sub = MySubRecord()
为 Avro schema 设置命名空间

使用特殊字段 _avro_namespace 设置Avro Record schema 的命名空间。

class NamespaceDemo(Record):
   _avro_namespace = 'xxx.xxx.xxx'
   x = String()
   y = Integer()

Schema定义示例如下

{
  'name': 'NamespaceDemo', 'namespace': 'xxx.xxx.xxx', 'type': 'record', 'fields': [
    {'name': 'x', 'type': ['null', 'string']}, 
    {'name': 'y', 'type': ['null', 'int']}
  ]
}

声明和验证 schema

您可以使用 BytesSchema, StringSchema, AvroSchema和 JsonSchema 之一，作为序列化方式发送消息。

在生产者创建时，Pulsar服务端将会校验对应topic中的序列化方式是否正确，并且生产者和服务端topic定义的序列化格式是兼容的 如果当前的生产者序列化格式于服务端存在的序列化格式不兼容，那么在创建生产者实例时将会抛出异常

一旦生产者通过确定的序列化格式创建后，他只接受该序列化格式的对象

同样，对于消费者或者reader，消费者将会获取具有特定序列化格式的消息，而不是原始的字节

示例

consumer = client.subscribe(
                  topic='my-topic',
                  subscription_name='my-subscription',
                  schema=AvroSchema(Example) )

while True:
    msg = consumer.receive()
    ex = msg.value()
    try:
        print("Received message a={} b={} c={}".format(ex.a, ex.b, ex.c))
        # Acknowledge successful processing of the message
        consumer.acknowledge(msg)
    except Exception:
        # Message failed to be processed
        consumer.negative_acknowledge(msg)

AvroSchema

您可以使用 BytesSchema 发送byte类型的数据。

示例

producer = client.create_producer(
                'bytes-schema-topic',
                schema=BytesSchema())
producer.send(b"Hello")

consumer = client.subscribe(

您可以使用 StringSchema 发送字符串类型的数据。

示例

producer = client.create_producer(
                'string-schema-topic',
                schema=StringSchema())
producer.send("Hello")

consumer = client.subscribe(
                'string-schema-topic',
                'sub',
                schema=StringSchema())
msg = consumer.receive()
str = msg.value()

您可以使用以下方法之一定义 AvroSchema 作为序列化类型

方法1：Record

您可以通过继承 Pulsar 中的 schema.Record 的类来声明 AvroSchema 类型，同时将字段定义为 类变量

示例

class Example(Record):
    a = Integer()
    b = Integer()

producer = client.create_producer(
                'avro-schema-topic',
                schema=AvroSchema(Example))
r = Example(a=1, b=2)
producer.send(r)

consumer = client.subscribe(
                'avro-schema-topic',
                'sub',
                schema=AvroSchema(Example))
msg = consumer.receive()
e = msg.value()

方法2：JSON定义

您可以使用 JSON 声明一个 AvroSchema 。在这种情况下，Avro schemas 是使用 JSON 定义的。

示例

下面是一个 使用一个 JSON 文件定义AvroSchema的例子 (company.avsc 定义）

{
    "doc": "this is doc",
    "namespace": "example.avro",
    "type": "record",
    "name": "Company",
    "fields": [
        {"name": "name", "type": ["null", "string"]},
        {"name": "address", "type": ["null", "string"]},
        {"name": "employees", "type": ["null", {"type": "array", "items": {
            "type": "record",
            "name": "Employee",
            "fields": [
                {"name": "name", "type": ["null", "string"]},
                {"name": "age", "type": ["null", "int"]}
            ]
        }}]},
        {"name": "labels", "type": ["null", {"type": "map", "values": "string"}]}
    ]
}

您可以使用 (http://avro.apache.org/docs/current/gettingstartedpython.html) 或 fastavro.schema文件, 来定义 [avro.schema] 类型的序列化方式

如果您使用 "JSON 定义" 定义 AvroSchema，请注意以下几点：

  • 您需要使用 Python dict 来生产消息和消费消息，而不是使用 "Record"

  • 当生成 AvroSchema 对象时，将 _record_cls 参数设置为 None

示例

from fastavro.schema import load_schema
from pulsar.schema import *
schema_definition = load_schema("examples/company.avsc")
avro_schema = AvroSchema(None, schema_definition=schema_definition)
producer = client.create_producer(
    topic=topic,
    schema=avro_schema)
consumer = client.subscribe(topic, 'test', schema=avro_schema)
company = {
    "name": "company-name" + str(i),
    "address": 'xxx road xxx street ' + str(i),
    "employees": [
        {"name": "user" + str(i), "age": 20 + i},
        {"name": "user" + str(i), "age": 30 + i},
        {"name": "user" + str(i), "age": 35 + i},
    ],
    "labels": {
        "industry": "software" + str(i),
        "scale": ">100",
        "funds": "1000000.0"
    }
}
producer.send(company)
msg = consumer.receive()
# Users could get a dict object by `value()` method.
msg.value()

您可以通过继承 pulsar.schema.Record 的类来声明 JsonSchema 类型，同时将字段定义为 类变量 这和使用 AvroSchema是类似的。 在你定于序列化类型时唯一区别是显示使用 JsonSchema 作为序列化类型而不是 AvroSchema作为序列化类型 如何通过Record使用 AvroSchema ，参见 这里。

producer = client.create_producer(
                'avro-schema-topic',
                schema=JsonSchema(Example))

consumer = client.subscribe(

端到端加密

端到端加密 可以让应用在生产端加密消息并在消费端解密消息。

配置

使用 Python 客户端的端到端加密功能，你需要为生产者和消费者配置 publicKeyPath 和 privateKeyPath。

publicKeyPath: "./public.pem"
privateKeyPath: "./private.pem"

教程

This section provides step-by-step instructions on how to use the end-to-end encryption feature in the Python client.

前提条件

  • Pulsar Python 客户端为 2.7.1或更高版本

步骤

  1. 创建公钥和私钥密钥对。

    输入

    openssl genrsa -out private.pem 2048
openssl rsa -in private.pem -pubout -out public.pem

  2. 创建一个生产者用来发送加密消息。

    输入

    import pulsar

publicKeyPath = "./public.pem"
privateKeyPath = "./private.pem"
crypto_key_reader = pulsar.CryptoKeyReader(publicKeyPath, privateKeyPath)
client = pulsar.Client('pulsar://localhost:6650')
producer = client.create_producer(topic='encryption', encryption_key='encryption', crypto_key_reader=crypto_key_reader)
producer.send('encryption message'.encode('utf8'))
print('sent message')
producer.close()
client.close()

  3. 创建消费者接收加密消息。

    输入

    import pulsar

publicKeyPath = "./public.pem"
privateKeyPath = "./private.pem"
crypto_key_reader = pulsar.CryptoKeyReader(publicKeyPath, privateKeyPath)
client = pulsar.Client('pulsar://localhost:6650')
consumer = client.subscribe(topic='encryption', subscription_name='encryption-sub', crypto_key_reader=crypto_key_reader)
msg = consumer.receive()
print("Received msg '{}' id = '{}'".format(msg.data(), msg.message_id()))
consumer.close()
client.close()

  4. 运行消费者接收加密消息。

    输入

    python consumer.py

  5. 在一个新的终端窗口中，运行生产者来生成加密消息。

    输入

    python producer.py

    现在你可以看到生产者发送消息，消费者成功收到消息。

    输出

    生产端：

     sent message

    消费端：

     Received msg 'encryption message' id = '(0,0,-1,-1)'
