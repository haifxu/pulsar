Set up a standalone Pulsar locally

對於本地開發和測試，你可以在自己的電腦上以單機模式來運行 Pulsar。 單機模式包含一個 Pulsar broker、必須的 Zookeeper 和 BookKeeper 組件，它們運行在一個單獨的 Java 虛擬機（JVM）進程之中。

Pulsar 運行在生產環境？

如果你想在生產環境運行完整的 Pulsar，請參考 Pulsar 的集群部署。

安裝 Pulsar 單機模式

本教程將講解安裝過程的每個步驟。

System requirements

Currently, Pulsar is available for 64-bit macOS, Linux, and Windows. To use Pulsar, you need to install 64-bit JRE/JDK 8 or later versions.

提示

By default, Pulsar allocates 2G JVM heap memory to start. It can be changed in conf/pulsar_env.sh file under PULSAR_MEM. This is extra options passed into JVM.

Note

Broker is only supported on 64-bit JVM.

使用二進制發行版安裝 Pulsar

要開始使用 Pulsar，請透過以下方式下載二進制 tarball 版本：

  • download from the Apache mirror (Pulsar 2.9.1 binary release)

  • 從 Pulsar 官網下載頁

  • 從 Pulsar 的 Github 發布頁面下載

  • 使用wget：

    $ wget https://archive.apache.org/dist/pulsar/pulsar-2.9.1/apache-pulsar-2.9.1-bin.tar.gz

下載 tarball 壓縮包後，解壓縮並使用cd命令切換 到結果目錄：

$ tar xvfz apache-pulsar-2.9.1-bin.tar.gz
$ cd apache-pulsar-2.9.1

What your package contains

The Pulsar binary package initially contains the following directories:

目錄包含
binPulsar's command-line tools, such as pulsar and pulsar-admin.
confPulsar 的配置文件，包含 broker 配置,ZooKeeper 配置 等等。
examples一個 Java JAR 包，包含 Pulsar Functions 的範例。
libThe JAR files used by Pulsar.
licenses開源許可文件 .txt格式, 適用於 Pulsar 代碼庫 裡的各種組件。

以下目錄是在運行 Pulsar 的時候生成。

目錄包含
dataThe data storage directory used by ZooKeeper and BookKeeper.
instances為 Pulsar Functions 創建的組件。
logsLogs created by the installation.

提示

If you want to use builtin connectors and tiered storage offloaders, you can install them according to the following instructions：

  • 安裝內建 Source/Sink 連接器（可選）
  • 安裝分層存儲卸載插件（可選）

Otherwise, skip this step and perform the next step Start Pulsar standalone. Pulsar can be successfully installed without installing bulitin connectors and tiered storage offloaders.

安裝內建 Source/Sink 連接器（可選）

自從2.1.0-incubating 版本，Pulsar 會為不同的 Source/Sink 連接器發布了一個單獨的二進制版。你可依實際的需要去下載這些內建的連接器。 要啟用這些 內建 連接器，首先到下列的下載點去下載所需要的連接器：

  • download from the Apache mirror Pulsar IO Connectors 2.9.1 release

  • 從 Pulsar 官網下載頁

  • 從 Pulsar 的 Github 發布頁面下載

  • 使用wget：

    $ wget https://archive.apache.org/dist/pulsar/pulsar-2.9.1/connectors/{connector}-2.9.1.nar

下載完 nar 文件後，將文件複製到 pulsar 目錄中的 connectors 目錄。 例如，如果你下載 pulsar-io-aerospike-2.9.1.nar 連接器檔案，請輸入以下命令：

$ mkdir connectors
$ mv pulsar-io-aerospike-2.9.1.nar connectors

$ ls connectors
pulsar-io-aerospike-2.9.1.nar
...

Note

  • If you are running Pulsar in a bare metal cluster, make sure connectors tarball is unzipped in every pulsar directory of the broker (or in every pulsar directory of function-worker if you are running a separate worker cluster for Pulsar Functions).

  • If you are running Pulsar in Docker or deploying Pulsar using a docker image (e.g. K8S or DC/OS), you can use the apachepulsar/pulsar-all image instead of the apachepulsar/pulsar image. apachepulsar/pulsar-all image has already bundled all builtin connectors.

安裝分層存儲卸載插件（可選）

提示

Since 2.2.0 release, Pulsar releases a separate binary distribution, containing the tiered storage offloaders. To enable tiered storage feature, follow the instructions below; otherwise skip this section.

要開始使用 分層存儲卸載插件，你需要透過以下任一種方式在每個 broker 節點安裝這個插件：

  • download from the Apache mirror Pulsar Tiered Storage Offloaders 2.9.1 release

  • 從 Pulsar 官網下載頁

  • 從 Pulsar 的 Github 發布頁面下載

  • 使用wget：

    $ wget https://archive.apache.org/dist/pulsar/pulsar-2.9.1/apache-pulsar-offloaders-2.9.1-bin.tar.gz

下載完壓縮包後，解壓縮並將"分層存儲卸載插件"複製到 pulsar 目錄中的offloaders目錄：

$ tar xvfz apache-pulsar-offloaders-2.9.1-bin.tar.gz

// you will find a directory named `apache-pulsar-offloaders-2.9.1` in the pulsar directory
// then copy the offloaders

$ mv apache-pulsar-offloaders-2.9.1/offloaders offloaders

$ ls offloaders
tiered-storage-jcloud-2.9.1.nar

有關如何配置分層存儲的更多內容，請參見 分層存儲指南。

Note

  • If you are running Pulsar in a bare metal cluster, make sure that offloaders tarball is unzipped in every broker's pulsar directory.

  • If you are running Pulsar in Docker or deploying Pulsar using a docker image (e.g. K8S or DC/OS), you can use the apachepulsar/pulsar-all image instead of the apachepulsar/pulsar image. apachepulsar/pulsar-all image has already bundled tiered storage offloaders.

啟動 Pulsar 單機模式

取得 Pulsar 發行版的最新本地副本後，可以使用 pulsar 命令，該命令位於bin目錄中，並且使用參數 standlane 指定要以單機模式啟動 Pulsar。

$ bin/pulsar standalone

如果成功啟動了 Pulsar，將看到如下所示的INFO級日誌消息：

2017-06-01 14:46:29,192 - INFO  - [main:WebSocketService@95] - Configuration Store cache started
2017-06-01 14:46:29,192 - INFO  - [main:AuthenticationService@61] - Authentication is disabled
2017-06-01 14:46:29,192 - INFO  - [main:WebSocketService@108] - Pulsar WebSocket Service started

提示

  • The service is running on your terminal, which is under your direct control. If you need to run other commands, open a new terminal window.
    You can also run the service as a background process using the pulsar-daemon start standalone command. For more information, see pulsar-daemon.

  • By default, there is no encryption, authentication, or authorization configured. 因此無需授權即可從遠端來訪問 Apache Pulsar。 請詳閱安全概述文檔來保護所部署的 Pulsar。

  • When you start a local standalone cluster, a public/default namespace is created automatically. 這個預設的命名空間主要用於開發。 All Pulsar topics are managed within namespaces. 更多相關內容请参考 Topics。

Pulsar 單機模式的操作

Pulsar 提供一個名為 pulsar-client 的 CLI 工具。 Pulsar-client 工具能夠在運行的 Pulsar 集群中對某一個指定topic來進行信息的發佈與消費。

消費一條信息

以下命令會從 my-topic 主題裡消費一個信息，訂閱名稱為 first-subscription ：

$ bin/pulsar-client consume my-topic -s "first-subscription"

如果topic訂閱成功，應該會在 pulsar-client 中看到類似以下的日誌：

09:56:55.566 [pulsar-client-io-1-1] INFO  org.apache.pulsar.client.impl.MultiTopicsConsumerImpl - [TopicsConsumerFakeTopicNamee2df9] [first-subscription] Success subscribe new topic my-topic in topics consumer, partitions: 4, allTopicPartitionsNumber: 4

提示

As you have noticed that we do not explicitly create the my-topic topic, to which we consume the message. When you consume a message to a topic that does not yet exist, Pulsar creates that topic for you automatically. Producing a message to a topic that does not exist will automatically create that topic for you as well.

產生一條信息

以下命令將發送一條簡單的信息 hello-pulsar 給 my-topic 主題:

$ bin/pulsar-client produce my-topic --messages "hello-pulsar"

如果信息成功發送給了 topic，應該會在 pulsar-client 中看到類似以下的日誌：

13:09:39.356 [main] INFO  org.apache.pulsar.client.cli.PulsarClientTool - 1 messages successfully produced

停止 Pulsar 單機模式

按 Ctrl+C 停止單機模式運行的 Pulsar。

提示

If the service runs as a background process using the pulsar-daemon start standalone command, then use the pulsar-daemon stop standalone command to stop the service.

For more information, see pulsar-daemon.
