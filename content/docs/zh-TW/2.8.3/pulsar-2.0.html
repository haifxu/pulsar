Pulsar 2.0

Pulsar 2.0 是 Pulsar 的新發布版本，為平台帶來了重大的改變，包括[簡化的 Topic 名字](#Topic 名字)，以及 Pulsar Functions 功能，同時還有一些術語的變化等等。

Pulsar 2.0 新功能

功能Description
Pulsar FunctionsPulsar的一個輕量級的 Function 運算功能

主要改變

你需要注意幾個主要改變，它們可能會對日常使用 Pular 的開發運維產生重大的影響。

Properties 對比 tenants

以前，Pulsar 有一個 properties（屬性）的概念。 A property is essentially the exact same thing as a tenant, so the "property" terminology has been removed in version 2.0. The pulsar-admin properties command-line interface, for example, has been replaced with the pulsar-admin tenants interface. 在某些情況下 properties 術語仍然有效，但是未來版本也會慢慢棄用。

Topic 名字

Prior to version 2.0, all Pulsar topics had the following form:

{persistent|non-persistent}://property/cluster/namespace/topic

2.0 版有兩個重要改變：

  • 移除 cluster 組件
  • property 組件被更名為 tenant
  • 你可以使用[彈性的 Topic 命名](#彈性的 Topic 命名)規則來縮短 Topic 名字。
  • / is not allowed in topic name

移除 cluster 組件

The cluster component has been removed from topic names. Thus, all topic names now have the following form:

{persistent|non-persistent}://tenant/namespace/topic

使用了舊的命名方式的topics，可以繼續運行而不需要特別的修改。

彈性的 Topic 命名

在 Pulsar 2.0 內部，所有 Topic 名字形態[如上](#移除 cluster 組件)，但是你現在可以通過簡化來使用更簡潔的名字。 彈性的命名系統有預設的 topic 類型，tenant 以及 namesapce：

Topic命名組件Default
topic 類型persistent
tenantpublic
namespacedefault

下表展示了一些topic名字預設自動轉換的例子：

輸入topic名字預設轉換成完整topic名字
my-topicpersistent://public/default/my-topic
my-tenant/my-namespace/my-topicpersistent://my-tenant/my-namespace/my-topic

對於非持久化 (non-persistent) 的 topic，你總是需要使用完整的 topic 名字，此時上面那種對於持久化 (persistent) topic 名字的轉換規則就不適用了。 這樣你不能適用 topic 短名如： non-persistent://my-topic ，而需要用non-persistent://public/default/my-topic 来宣告了。
