Pulsar Functions command line tool

下方圖表列出的是 Pulsar Functions command-line 工具。 你可以進行學習/參考這些 Pulsar Function 模式、命令和參數。

localrun

本地運行 Pulsar Function ，而不是將其部署到 Pulsar 集群。

NameDescriptionDefault
auto-ack是否自動確認信息。true
broker-service-urlPulsar broker 的 URL。
classnamePulsar Function 的類別名稱。
client-auth-params客戶端身份驗證參數。
client-auth-plugin客戶端身份驗證插件使用哪個函數進程可以連接到 broker。
CPU需要分配給每個函數實例 (僅適用於 broker 運行時間) 的 CPU 核心數。
custom-schema-inputs輸入 topic 對應到 Schema 類別名稱的映射（使用 JSON 字串）。
custom-serde-inputs輸入 topic 對應到 SerDe 類別名稱的映射（使為 JSON 字串）。
dead-letter-topic發送所有未成功處理的信息的 topic。 This parameter is not supported in Python Functions.
disk每個 function 實例需要分配的磁盤 (以 byte 為單位)空間 (僅適用使用 docker 來運行時)。
fqfnPulsar Function 的 Fully Qualified Function Name (FQFN)
function-config-file指定配置 Pulsar Function 的 YAML 設定檔路徑。
go執行 Pulsar Function 所需要的 Go executable binary 的路徑（如果該函數是用 Go 編寫的）。 It also supports URL path [http/https/file (file protocol assumes that file already exists on worker host)/function (package URL from packages management service)] from which worker can download the package.
hostname-verification-enabled啟用 hostname 驗證。false
inputsPulsar Function 的一個或多個輸入 topic（可以使用逗號分隔來指定多個 topic）。
jar該函數的 jar 文件的路徑（如果該函數是用Java編寫的）。 It also supports URL path [http/https/file (file protocol assumes that file already exists on worker host)/function (package URL from packages management service)] from which worker can download the package.
instance-id-offset指定特定 instance-id 啟動的 offset。0
log-topicPulsar Function 日誌所使用的日誌 topic。
max-message-retries在放棄處理一個信息之前應該要嘗試幾次。
namePulsar Function 的名稱
namespacePulsar Function 的命名空間
outputPulsar Function 的輸出 topic（如果沒有指定，就沒有輸出拋出）。
output-serde-classname該 function 用於輸出信息的 SerDe 類別。
parallelismPulsar Function 的並行度因子（即要運行的 function 實例的數量）。
processing-guarantees要套用在 function 信息的處理保證。 可用選項有： [ATLEAST_ONCE, ATMOST_ONCE, EFFECTIVELY_ONCE]。ATLEAST_ONCE
py該 function 的主要 Python 文件/ Python Wheel 文件的路徑（如果該 function 是用 Python 編寫的）。 It also supports URL path [http/https/file (file protocol assumes that file already exists on worker host)/function (package URL from packages management service)] from which worker can download the package.
ram需要為每個 function 實例分配的記憶體 (byte) 用量（僅適用於 process/docker 運行環境）。
retain-orderingFunction 按順序使用和處理信息。
schema-type用於輸出信息的內建 schema 資料類型或自定義 schema 類別名稱。
sliding-interval-count窗口滑動之後的信息數。
sliding-interval-duration-ms窗口滑動之後的時間長度。
subs-name指定 Pulsar input-topic 消費者的 subscription 名稱
tenantPulsar Function 的租戶名稱
timeout-ms消息超時 (以毫米計)。
tls-allow-insecure允許不安全的 tls 連接。false
tls-trust-cert-pathtls 信任證書文件路徑。
topics-pattern消費 topic 的匹配模版。 [--input] 與 [--topic-pattern] 是互斥的選項。 可以在 --custom-serde-inputs 指定 SerDe 類別名稱 (僅支援 Java 開發的 function)。
use-tls使用 tls 連接false
user-config用戶定義的配置 key/value 對。
window-length-count每個窗口的信息數。
window-length-duration-ms每個窗口的時間長度（以毫秒為單位）。

user-config用戶定義的配置 key/value 對。
