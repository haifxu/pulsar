Pulsar adaptor for Apache Kafka

Pulsar est compatible avec les applications qui utilisent l'API Java d'Apache Kafka.

À l’aide du wrapper de compatibilité Pulsar pour Kafka

In an existing application, change the regular Kafka client dependency and replace it with the Pulsar Kafka wrapper. Remove the following dependency in pom.xml:

<dependency>
  <groupId>org.apache.kafka</groupId>
  <artifactId>kafka-clients</artifactId>
  <version>0.10.2.1</version>
</dependency>

Ensuite, incluez la dépendance suivante pour le wrapper Pulsar pour Kafka :

<dependency>
  <groupId>org.apache.pulsar</groupId>
  <artifactId>pulsar-client-kafka</artifactId>
  <version>2.9.1</version>
</dependency>

Avec la nouvelle dépendance, le code existant fonctionne sans aucun changement. Enfin, vous devrez ajuster la configuration pour vous assurer que les producteurs et les consommateurs pointent sur Pulsar plutôt que Kafka, et utilisent un topic particulier de Pulsar.

Utiliser le wrapper de compatibilité Pulsar pour Kafka avec un client existant de Kafka

Lors de la migration de Kafka à Pulsar, l’application peut avoir à utiliser le client original Kafka et le wrapper Pulsar pour Kafka ensemble pendant la migration. Vous devriez envisager d'utiliser le wrapper client de Pulsar pour Kafka unshaded.

<dependency>
  <groupId>org.apache.pulsar</groupId>
  <artifactId>pulsar-client-kafka-original</artifactId>
  <version>2.9.1</version>
</dependency>

Lorsque vous utilisez cette dépendance, construisez des producteurs utilisant org.apache.kafka.clients.producer.PulsarKafkaProducer au lieu de org.apache.kafka.clients.producer.KafkaProducer et org.apache.kafka.clients.producer.PulsarKafkaConsumer pour les consommateurs.

Example de producteur

// Le topic doit être un topic classique de Pulsar
String topic = "persistent://public/default/my-topic";

Properties props = new Properties();
// Référence le service Pulsar
props.put("bootstrap.servers", "pulsar://localhost:6650");

props.put("key.serializer", IntegerSerializer.class.getName());
props.put("value.serializer", StringSerializer.class.getName());

Producer<Integer, String> producer = new KafkaProducer<>(props);

for (int i = 0; i < 10; i++) {
    producer.send(new ProducerRecord<Integer, String>(topic, i, "hello-" + i));
    log.info("Message {} sent successfully", i);
}

producer.close();

Exemple de consommateur

String topic = "persistent://public/default/my-topic";

Properties props = new Properties();
// Référence le service Pulsar
props.put("bootstrap.servers", "pulsar://localhost:6650");
props.put("group.id", "my-subscription-name");
props.put("enable.auto.commit", "false");
props.put("key.deserializer", IntegerDeserializer.class.getName());
props.put("value.deserializer", StringDeserializer.class.getName());

Consumer<Integer, String> consumer = new KafkaConsumer<>(props);
consumer.subscribe(Arrays.asList(topic));

while (true) {
    ConsumerRecords<Integer, String> records = consumer.poll(100);
    records.forEach(record -> {
        log.info("Received record: {}", record);
    });

    // Commit le dernier offset
    consumer.commitSync();
}

Personnaliser les configurations de Pulsar

Vous pouvez configurer le fournisseur d'authentification de Pulsar directement à partir des propriétés Kafka.

Propriétés du client Pulsar

PropriétéDefaultRemarques
pulsar.authentication.classConfigure to auth provider. For example, org.apache.pulsar.client.impl.auth.AuthenticationTls.
pulsar.authentication.params.mapMap qui représente les paramètres pour le plugin d'authentification.
pulsar.authentication.params.stringString avec les paramètres pour le plugin d'authentification, par exemple, key1:val1,key2:val2.
pulsar.use.tlsfalseActive la sécurité de la couche transport avec TLS.
pulsar.tls.trust.certs.file.pathChemin d'accès au fichier contenant les certificats de confiance TLS.
pulsar.tls.allow.insecure.connectionfalseAccepte les certificats auto-signés des brokers.
pulsar.operation.timeout.ms30000Délai d'attente général des opérations.
pulsar.stats.interval.seconds60Intervalle d'émission des statistiques de la lib cliente Pulsar.
pulsar.num.io.threads1Le nombre de threads Netty IO à utiliser.
pulsar.connections.per.broker1Le nombre maximum de connexions à chaque broker.
pulsar.use.tcp.nodelaytrueTCP no-delay.
pulsar.concurrent.lookup.requests50000Le nombre maximum de recherches simultanées sur les topics.
pulsar.max.number.rejected.request.per.connection50Le seuil d'erreurs pour forcer la fermeture d'une connexion.
pulsar.keepalive.interval.ms30000Keep alive interval for each client-broker-connection.
